¿Qué es ECMAScript?

ECMAScript es un estándar de lenguaje de programación desarrollado por ECMA International. Sirve como base para lenguajes como JavaScript, JScript y ActionScript. Fue creado para estandarizar el lenguaje JavaScript, permitiendo que todos los navegadores lo interpreten de la misma forma.

¿Cuáles son las versiones de ECMAScript?

ES1 (1997)
Primera versión.
Sintaxis básica, operadores, estructuras de control, funciones.

ES2 (1998)
Ajustes menores para alinearse con estándares ISO.
ES3 (1999)
Agrega expresiones regulares, estructuras try/catch para manejo de errores, operadores new/delete, y bucle do-while.
ES4 (cancelada)
Propuesta muy ambiciosa con muchos cambios; fue cancelada en 2008.
ES5 (2009)
Introduce el modo estricto, funciones como JSON.parse, métodos como forEach, map, filter en arrays, y Object.create.
Ejemplo: Se activa el modo estricto, se define un objeto persona con nombre y edad, y se imprime su propiedad nombre.
ES6 / ES2015
Introduce let y const para declarar variables.
Agrega funciones flecha, clases, plantillas literales con backticks, promesas y módulos.
Ejemplo: Se declara una variable x con valor 5 y una constante y con valor 10. Luego, se define una función flecha que suma dos valores y se imprime el resultado de sumar x e y.
ES7 / ES2016
Añade el método includes para arrays y el operador exponencial (**).
Ejemplo: Se verifica si el número 2 está en un array y se calcula 2 elevado a la 3.
ES8 / ES2017
Introduce async/await para asincronía, Object.entries y Object.values.
Ejemplo: Se define una función asíncrona que retorna un saludo y se imprime su resultado usando then.
ES9 / ES2018
Permite el uso de Rest/Spread en objetos y agrega finally a las promesas.
Ejemplo: Se copia un objeto usando el operador spread y se le añade una nueva propiedad.
ES10 / ES2019
Introduce Array.flat para aplanar arrays anidados, Object.fromEntries y métodos para eliminar espacios en cadenas.
Ejemplo: Se aplana un array con un subarray.
ES11 / ES2020
Incorpora el operador nullish coalescing (??), el encadenamiento opcional (?.), y Promise.allSettled.
Ejemplo: Se evalúa una variable nula y se imprime un valor por defecto si es null.
ES12 / ES2021
Añade operadores de asignación lógica (&&=, ||=, ??=) y el método replaceAll.
Ejemplo: Se reemplazan todas las ocurrencias de una palabra por otra en una cadena.
ES13 / ES2022
Introduce campos privados en clases usando # y el método at() en arrays para acceder por índice (incluidos negativos).
Ejemplo: Se define una clase Persona con nombre privado y un método para saludar.
ES14 / ES2023
Añade Array.findLast y Array.findLastIndex para encontrar el último elemento que cumple una condición.
Ejemplo: Se encuentra el último número menor que 4 en un array.
ES15 / ES2024 (propuesta)
En discusión. Se espera una mejora en interoperabilidad entre módulos, mejoras de tipos, y estructuras más declarativas.

Ejemplos de ECMAScript por version

ES5 (2009)
"use strict"; var persona = {   nombre: "Ana",   edad: 30 }; console.log(persona.nombre); // "Ana"

ES6 / ES2015
let x = 5; const y = 10; const sumar = (a, b) => a + b; console.log(sumar(x, y)); // 15
ES7 / ES2016
const numeros = [1, 2, 3]; console.log(numeros.includes(2)); // true console.log(2 ** 3); // 8
ES8 / ES2017
async function saludar() {   return "Hola"; } saludar().then(console.log); // "Hola"
ES9 / ES2018
const original = { nombre: "Luis" }; const copia = { ...original, edad: 25 }; console.log(copia); // { nombre: "Luis", edad: 25 }
ES10 / ES2019
const anidado = [1, [2, 3]]; console.log(anidado.flat()); // [1, 2, 3]
ES11 / ES2020
const usuario = null; console.log(usuario ?? "Invitado"); // "Invitado"
ES12 / ES2021
let mensaje = "Hola mundo mundo"; console.log(mensaje.replaceAll("mundo", "a todos")); // "Hola a todos a todos"
ES13 / ES2022 class Persona {   #nombre = "Carlos";   saludar() {     console.log(Hola, soy ${this.#nombre});   } } new Persona().saludar(); // "Hola, soy Carlos"
ES14 / ES2023
const numeros = [1, 3, 5, 2, 4]; const ultimoMenorQue4 = numeros.findLast(n => n < 4); console.log(ultimoMenorQue4); // 2

¿Cómo utilizar ECMAScript?

ECMAScript se utiliza principalmente a través del lenguaje JavaScript en navegadores modernos o entornos como Node.js. No requiere configuración especial, ya que basta con escribir código JavaScript en un editor de texto y ejecutarlo en un navegador actualizado o mediante la consola con Node.js. Las funciones de ECMAScript están disponibles según la versión que soporte el entorno en uso, por lo que para aprovechar características recientes es importante trabajar en plataformas actualizadas. En desarrollo web, los archivos JavaScript se enlazan a documentos HTML, mientras que en entornos de servidor se ejecutan directamente. Si se necesita compatibilidad con navegadores antiguos, se pueden usar herramientas como Babel que convierten el código moderno a versiones anteriores.

¿Cuáles son las diferencias entre versiones?

ECMAScript ha evolucionado constantemente para adaptarse a las necesidades del desarrollo moderno. Las diferencias clave entre versiones reflejan el paso de un lenguaje básico a uno altamente expresivo y modular. ES3 fue la base que introdujo estructuras más completas como try/catch, pero fue con ES5 donde el lenguaje comenzó a madurar con herramientas fundamentales como el modo estricto y funciones para manipular arrays y objetos. La gran transformación llegó con ES6 (2015), que revolucionó JavaScript con clases, módulos, funciones flecha, let/const, y promesas, marcando un antes y un después en la sintaxis moderna. Las versiones posteriores (ES7 a ES9) refinaron el lenguaje, agregando operadores útiles como exponentiation y métodos de arrays más expresivos. Con ES8, la asincronía se volvió más intuitiva mediante async/await, mientras que ES9 mejoró el trabajo con objetos y promesas. ES10 y ES11 continuaron la tendencia de simplificación, añadiendo herramientas como flat(), encadenamiento opcional y nullish coalescing para evitar errores comunes. ES12 introdujo operadores de asignación lógica que permiten escribir condiciones más limpias y eficientes. ES13 y ES14 consolidaron el uso de clases con campos privados y mejoraron la manipulación de arrays. En conjunto, estas diferencias muestran una evolución hacia un lenguaje más potente, conciso, seguro y accesible para desarrolladores de todos los niveles.
